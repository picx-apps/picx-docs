图床配置 ​

PicX 的图床服务依托于 GitHub，在上传图片之前，你需要先使用 GitHub Token 来完成 仓库、分支 和 目录 之间的绑定。

  • 仓库 - 托管图片的图床
  • 目录 - 存放图片的文件夹

GitHub Token ​

PicX 基于 GitHub API 实现图片的上传、删除和管理，你必须拥有一个 GitHub Token，才能获得 PicX 的图床服务。

获取 GitHub Token ​

创建 一个带有 repo 权限的 GitHub Token。

点击 https://github.com/settings/tokens/new 快速新建 GitHub Token

点击 Generate token 按钮，生成 GitHub Token。

注意

新生成的 Token 只会显示一次，请妥善保存，如有遗失，重新生成即可。

绑定 GitHub Token ​

通过 GitHub Token 可以获取到你的 GitHub 用户信息、Public 仓库信息等数据。

绑定仓库 ​

在 PicX 中，GitHub 仓库就是图床，上传的所有图片都存储在仓库里，你可以使用一个已有的仓库，也可以新建一个仓库专门用作图床（推荐）。

推荐使用 一键自动配置，自动创建仓库并完成仓库、分支和目录之间的绑定。

新建仓库 ​

创建 一个用来存储图片的 GitHub 仓库，仓库类型必须选 Public 。

点击链接 https://github.com/new 快速新建仓库

选择仓库 ​

在选择仓库的下拉列表中，选择刚刚新建的仓库作为图床。

绑定分支 ​

如果你绑定的仓库有多个分支，则会出现选择分支的下拉列表，选择其中一个即可。

暂时不支持通过 PicX 来给 GitHub 仓库新建分支，如果你想要多个分支，必须手动去创建。

然而，PicX 支持创建多级目录，多分支可有可无，如果你不熟悉 GitHub 仓库的分支创建，完全可以忽略这一步。

绑定目录 ​

在 PicX 中，目录是指仓库里存放图片的文件夹，上传图片之前，需绑定一个目录。

  • 新建目录：手动输入一个新目录，图片上传时会自动创建。

    使用 / 可以创建多级目录，最大支持 5 级。
    例如：x01/x02/x03，表示创建到了 3 级目录。

  • 根目录：图片将存储在仓库根目录下。

  • 自动目录：自动生成日期格式 yyyyMMdd 的目录名，例如：20230605。

  • 仓库目录

    如果你绑定的仓库和分支下面存在多个目录，选择其中一个即可。

至此，完成手动配置图床，点击 确定 按钮即可跳转到上传图片。
