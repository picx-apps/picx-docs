图床配置 ​

使用 PicX 进行上传图片之前，你需要先完成 GitHub Token、仓库、分支和目录（存放图片的文件夹）之间的绑定。

GitHub Token ​

PicX 基于 GitHub API 实现图片的上传、删除和管理，你必须拥有一个 GitHub Token，才能使用 PicX 图床的全部功能。

获取 GitHub Token ​

创建 一个带有 repo 权限的 GitHub Token 。

点击链接 https://github.com/settings/tokens/new 快速创建 Token

注意

新生成的 Token 只会显示一次，请妥善保存，如有遗失，重新生成即可

绑定 GitHub Token ​

通过 GitHub Token 即可获取到 GitHub 用户信息、仓库信息、分支信息和目录信息。

绑定仓库 ​

在 PicX 中，GitHub 仓库就是图床，所有上传的图片都存储在仓库里，你可以使用一个现有的仓库，也可以新建一个仓库（推荐）。

新建仓库 ​

创建 一个用来存储图片的 GitHub 仓库，仓库属性必须选 Public 。

点击链接 https://github.com/new 快速新建仓库

选择仓库 ​

在选择仓库的下拉列表中，选择刚刚新建的仓库作为图床。

绑定分支 ​

如果你绑定的仓库有多个分支，那么会出现选择分支的下拉列表，否则直接进入选择目录。

  • 多个分支情况，选择其中一个即可。

    PicX 暂时不支持新建分支，需要你手动去创建。

  • 只有一个分支情况，无分支下拉列表。

绑定目录 ​

在 PicX 中，目录是指仓库里存放图片的文件夹，上传图片之前，需绑定一个目录。

  • 新建目录：手动输入一个新目录，图片上传时会自动创建。

  • 根目录：图片将存储在仓库根目录下。

  • 自动目录：自动生成日期格式 yyyyMMdd 的目录名，例如：20230320。

如果你绑定的仓库和分支下面已经存在多个目录，那么你可以直接选择其中一个。

至此，完成图床配置，点击 “确认” 按钮即可跳转到图片上传界面。
